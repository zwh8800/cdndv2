# 组合型 API 重构设计

## 一、问题分析

当前系统有 **79 个工具**分布在 **11 个 SubAgent** 中，存在以下核心问题：

1. **工具过于细粒度**：几乎 1:1 映射 dnd-core API，如创建角色需要 LLM 先查种族、查职业、再调 create_pc
2. **Agent 过于碎片化**：movement_agent(5个工具)、mount_agent(3个工具)、npc_agent(2个工具) 等 Agent 职责过小
3. **delegate_task 只列了 3 个 Agent**（character/combat/rules），但实际有 11 个
4. **LLM 选择困难**：MainAgent 暴露 40 个只读工具 + delegate_task，工具爆炸

## 二、设计原则

- 按 D&D 游戏流程组织，而非按数据模型组织
- 组合工具封装完整业务流程，一次调用完成一个有意义的游戏操作
- 底层工具保留但不直接暴露给 MainAgent，仅供组合工具内部调用或 SubAgent 使用
- 新工具文件放在 `game_engine/tool/composite_*.go`，与现有工具共存
- SubAgent 精简从 11 个合并到 5 个，每个 Agent 职责清晰

## 三、SubAgent 精简方案（11 -> 5）

| 新 Agent | 合并自 | 职责 |
|----------|--------|------|
| `character_agent` | character + data_query | 角色创建、升级、数据查询 |
| `combat_agent` | combat + movement + mount | 战斗、移动、坐骑 |
| `rules_agent` | rules | 检定、法术、休息 |
| `world_agent` | narrative + npc + memory + crafting | 场景、NPC、任务、探索、制作 |
| `inventory_agent` | inventory (含 magic_item) | 物品管理、装备、魔法物品 |

data_query_agent 的工具直接分配给需要的 Agent（角色数据给 character_agent，法术数据给 rules_agent 等）。

## 四、组合工具设计（共 23 个）

---

### 域 1：角色管理 - `composite_character_tools.go`

**1. `create_player_character`（写）**
- 描述：一站式创建完整的玩家角色，自动验证种族/职业合法性，创建成功后自动推进游戏阶段
- 封装：GetRace + GetClass + GetBackground + CreatePC + SetPhase(自动)
- 参数：`game_id, name, race(中文), class(中文), ability_scores{6项}, background(可选), alignment(可选), level(默认1)`
- 内部逻辑：验证种族/职业 -> 创建角色 -> 若 phase=character_creation 则自动切到 exploration

**2. `spawn_creature`（写）**
- 描述：创建 NPC、敌人或同伴，统一入口，通过 creature_type 区分。支持从怪物模板快速创建
- 封装：CreateNPC / CreateEnemy / CreateCompanion + GetMonster(可选)
- 参数：`game_id, name, creature_type(enum: npc/enemy/companion), monster_template(可选), description(可选), hit_points(可选), armor_class(可选), ability_scores(可选), attitude(可选,仅NPC), leader_id(可选,仅companion)`

**3. `query_character`（读）**
- 描述：统一角色信息查询，支持单个查询、列表查询、PC详情查询
- 封装：GetActor / GetPC / ListActors
- 参数：`game_id, query_type(enum: single/list/pc_detail), actor_id(可选), type_filter(可选)`

---

### 域 2：战斗管理 - `composite_combat_tools.go`

**4. `initiate_combat`（写）**
- 描述：发起战斗遭遇，可同时创建敌人并开始战斗，自动切换到 combat 阶段。支持突袭
- 封装：CreateEnemy(批量,可选) + StartCombat/StartCombatWithSurprise + SetPhase(combat)
- 参数：`game_id, scene_id, participant_ids, enemies_to_create(可选,数组), surprise(可选)`

**5. `combat_action`（写）**
- 描述：执行当前回合的战斗动作，统一所有战斗操作
- 封装：ExecuteAttack / ExecuteAction / CastSpell / MoveActor / ExecuteDamage / ExecuteHealing
- 参数：`game_id, actor_id, action(enum: attack/cast_spell/dash/disengage/dodge/help/hide/ready/search/move/damage/heal), target_id(可选), weapon_id(可选), spell_id(可选), target_ids(可选), slot_level(可选), x(可选), y(可选), amount(可选), damage_type(可选), advantage(可选), auto_next_turn(可选,默认false)`

**6. `resolve_combat`（写）**
- 描述：结束战斗并处理善后（分配经验值、切换回 exploration 阶段）
- 封装：EndCombat + AddExperience(批量,可选) + SetPhase(exploration)
- 参数：`game_id, xp_per_character(可选)`

**7. `query_combat`（读）**
- 描述：查询当前战斗状态和回合信息
- 封装：GetCurrentCombat + GetCurrentTurn
- 参数：`game_id, include_turn_info(可选,默认true)`

---

### 域 3：规则与检定 - `composite_rules_tools.go`

**8. `perform_check`（写）**
- 描述：统一检定工具，支持属性检定、技能检定、豁免检定
- 封装：PerformAbilityCheck / PerformSkillCheck / PerformSavingThrow
- 参数：`game_id, actor_id, check_type(enum: ability/skill/saving_throw), ability(可选), skill(可选), dc(可选), advantage(可选), reason(可选)`

**9. `cast_spell`（写）**
- 描述：施放法术，自动处理专注（替换已有专注法术时自动结束旧专注）
- 封装：CastSpell + ConcentrationCheck(自动) + EndConcentration(自动)
- 参数：`game_id, caster_id, spell_id, target_ids(可选), slot_level(可选)`

**10. `manage_spells`（写）**
- 描述：管理法术（学习、准备），合并低频法术管理操作
- 封装：LearnSpell / PrepareSpells
- 参数：`game_id, caster_id, operation(enum: learn/prepare), spell_id(learn时), spell_ids(prepare时)`

**11. `take_rest`（写）**
- 描述：统一的休息工具，自动处理所有休息效果（HP恢复、法术位恢复、魔法物品充能）
- 封装：ShortRest / StartLongRest+EndLongRest + RechargeMagicItems + SetPhase
- 参数：`game_id, actor_ids, rest_type(enum: short/long)`

**12. `query_spell_status`（读）**
- 描述：查询法术位状态和被动感知
- 封装：GetSpellSlots + GetPassivePerception
- 参数：`game_id, actor_id, include_passive_perception(可选,默认false)`

---

### 域 4：世界与叙事 - `composite_world_tools.go`

**13. `setup_scene`（写）**
- 描述：一站式场景搭建（创建+设为当前+放置角色和物品+建立连接）
- 封装：CreateScene + SetCurrentScene + MoveActorToScene(批量) + AddItemToScene(批量) + AddSceneConnection(可选)
- 参数：`game_id, name, description, scene_type, set_as_current(默认true), actor_ids(可选), items(可选), connections(可选)`

**14. `npc_social_interaction`（写）**
- 描述：与NPC社交互动，自动获取角色属性进行检定（不再需要LLM手动传入ability值）
- 封装：GetActor(自动获取属性) + InteractWithNPC + GetNPCAttitude
- 参数：`game_id, actor_id(PC), npc_id, check_type(enum: persuasion/intimidation/deception/performance), advantage(可选)`

**15. `manage_quest`（写）**
- 描述：统一的任务管理工具
- 封装：CreateQuest / AcceptQuest / UpdateQuestObjective / CompleteQuest / FailQuest
- 参数：`game_id, operation(enum: create/accept/update_objective/complete/fail), quest_name(可选), quest_description(可选), giver_id(可选), giver_name(可选), objectives(可选), rewards(可选), quest_id(可选), objective_id(可选), progress(可选)`

**16. `travel`（写）**
- 描述：完整旅行流程（开始+推进+遭遇检定）
- 封装：StartTravel + AdvanceTravel + PerformEncounterCheck(可选)
- 参数：`game_id, destination, terrain, distance, pace(默认normal), hours_to_advance(可选), check_encounters(可选,默认true)`

**17. `query_world`（读）**
- 描述：统一世界信息查询
- 封装：GetScene / ListScenes / GetCurrentScene / GetSceneActors / GetSceneItems / GetNPCAttitude / ListQuests / GetQuest
- 参数：`game_id, query_type(enum: current_scene/scene_detail/scene_list/scene_actors/scene_items/npc_attitude/quest_list/quest_detail), scene_id(可选), npc_id(可选), quest_id(可选)`

---

### 域 5：物品管理 - `composite_inventory_tools.go`

**18. `manage_item`（写）**
- 描述：物品增删和转移的统一操作入口
- 封装：AddItem / RemoveItem / TransferItem + AddCurrency
- 参数：`game_id, actor_id, operation(enum: add/remove/transfer/add_currency), item(add时), item_id(remove/transfer时), quantity(可选), target_actor_id(transfer时), currency(add_currency时)`

**19. `equip_item`（写）**
- 描述：装备/卸下/调谐的统一操作，支持一键装备并调谐
- 封装：EquipItem / UnequipItem / AttuneItem / UnattuneItem
- 参数：`game_id, actor_id, item_id, operation(enum: equip/unequip/attune/unattune/equip_and_attune)`

**20. `use_item`（写）**
- 描述：使用物品（魔法物品、消耗品）
- 封装：UseMagicItem
- 参数：`game_id, actor_id, item_id, target_ids(可选)`

**21. `query_inventory`（读）**
- 描述：查询库存和装备信息
- 封装：GetInventory + GetEquipment + GetMagicItemBonus
- 参数：`game_id, actor_id, include_equipment(默认true), include_magic_bonuses(默认false)`

---

### 域 6：数据查询 - `composite_data_query_tools.go`

**22. `lookup_game_data`（读）**
- 描述：统一 D&D 规则数据查询，替代 16 个独立查询工具
- 封装：所有 List*/Get* 数据查询工具
- 参数：`data_type(enum: race/class/background/monster/spell/weapon/armor/magic_item/feat), name(查单个详情时), page(可选), page_size(可选)`

---

### 域 7：游戏管理 - 保留原有工具

**23. `set_phase` / `get_phase`**
- 保留原有的 phase 工具不变，多数情况由组合工具自动调用

## 五、MainAgent 工具暴露策略

重构后 MainAgent 只暴露以下 7 个工具（从 40 个大幅缩减）：

| 工具 | 类型 | 说明 |
|------|------|------|
| `delegate_task` | 写 | 委托给 5 个 SubAgent |
| `query_character` | 读 | 角色查询 |
| `query_combat` | 读 | 战斗状态查询 |
| `query_spell_status` | 读 | 法术位查询 |
| `query_world` | 读 | 世界信息查询 |
| `query_inventory` | 读 | 库存查询 |
| `lookup_game_data` | 读 | 规则数据查询 |

## 六、delegate_task 重构

更新 agent_name enum 为 5 个新 Agent：
```
- character_agent: 角色创建、升级、经验分配
- combat_agent: 战斗发起、战斗动作、战斗结束、移动
- rules_agent: 检定(属性/技能/豁免)、法术、休息
- world_agent: 场景搭建、NPC社交、任务管理、旅行探索
- inventory_agent: 物品管理、装备、使用物品
```
