# 统一种族名称格式修复计划

## 问题
- `create_pc` 接受 `"human"` 创建成功，但未校验种族有效性，导致角色存储了非法 race 值
- `get_race` 只能以中文（如 `"人类"`）精确查询， `"Human"` / `"human"` 均失败
- 输入输出格式不一致，LLM 容易出错

## 方案（更新：只支持中文标准名，通过提示词预制数据引导 LLM）

### 任务 1：dnd-core CreatePC 增加 race 有效性校验（`pkg/engine/actor.go`）

在 `CreatePC` 构建 `PlayerCharacter` 前：
- 调用 `data.GetRace(req.PC.Race)` 校验种族是否存在
- 若无效，返回 `fmt.Errorf("无效的种族: %s，可用种族: %s", req.PC.Race, strings.Join(data.GetRaceNames(), ", "))`
- 确保只有中文标准名能通过校验

这样非法值如 `"human"`、`"Human"` 都会被拒绝，LLM 必须使用正确的中文名。

### 任务 2：dnd-core 导出可用种族列表（`pkg/data/races.go`）

确保 `GetRaceNames()` 已导出并可被 engine 包使用，用于错误提示中列出可用种族。

### 任务 3：cdndv2 更新 create_pc 工具 schema（`game_engine/tool/character_tools.go`）

更新 `create_pc` 的 `race` 字段 JSON schema：
- `description` 改为明确的中文枚举说明："种族名称，必须使用以下中文标准名之一：人类、精灵、矮人、半身人、龙裔、侏儒、半精灵、半兽人、提夫林"
- 增加 `enum: ["人类", "精灵", "矮人", "半身人", "龙裔", "侏儒", "半精灵", "半兽人", "提夫林"]` 严格限制输入

同样处理 `class` 字段（职业），也应在 description 中明确中文标准名。

### 任务 4：cdndv2 character agent 提示词注入预制数据（`game_engine/prompt/character_system.md`）

在 character agent 的 system prompt 中注入系统预置的关键数据，让 LLM 明确知道应该使用哪些标准值：

- **种族列表**：人类、精灵（子种族：高等精灵、木精灵、卓尔）、矮人（子种族：丘陵矮人、山地矮人）、半身人（子种族：轻足、强魄）、龙裔、侏儒（子种族：森林侏儒、岩侏儒）、半精灵、半兽人、提夫林
- **职业列表**：野蛮人、吟游诗人、牧师、德鲁伊、战士、武僧、圣武士、游侠、游荡者、术士、邪术师、法师
- **背景列表**：侍僧、骗子、罪犯、艺人、平民英雄、公会工匠、隐士、贵族、智者、水手、士兵、流浪儿
- **个性特征模板**：提供标准 bonds/ideals/flaws/traits 的生成指引

这样 character agent 在调用 `create_pc` 时，自然会使用系统预置的中文标准名称。

### 任务 5：cdndv2 更新 get_race 工具描述（`game_engine/tool/data_query_tools.go`）

更新 `get_race` 的 `name` 字段 description：
- 明确说明必须使用中文标准名查询（如："人类"）

### 任务 6：验证编译

- 分别对 `dnd-core` 和 `cdndv2` 执行 `go build ./...` 确保编译通过

## 涉及文件

| 项目 | 文件路径 | 修改类型 |
|------|---------|---------|
| dnd-core | `pkg/engine/actor.go` | `CreatePC` 中增加 race 校验 |
| dnd-core | `pkg/data/races.go` | 确保 `GetRaceNames` 可用 |
| cdndv2 | `game_engine/tool/character_tools.go` | race/class 字段加 enum + 更新描述 |
| cdndv2 | `game_engine/tool/data_query_tools.go` | 更新 get_race 描述 |
| cdndv2 | `game_engine/prompt/character_system.md` | 注入种族/职业/背景预制数据 |

## 预期效果
- `create_pc("human")` → **失败**，返回 `"无效的种族: human，可用种族: 人类, 精灵, 矮人..."`
- `create_pc("人类")` → 成功
- `get_race("人类")` → 成功返回中文种族信息
- `list_races` 返回的名称始终为中文
- character agent 的 prompt 中已预制标准数据，LLM 会自觉使用中文名调用工具
