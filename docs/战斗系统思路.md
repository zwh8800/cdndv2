LLM-DnD 战斗系统架构设计

1. 系统核心思路

设计原则：战斗系统与探索叙事解耦，追求极简API（你提到"API越少越精准"），让Agent专注战斗逻辑本身。

---

2. Agent 分工

Agent职责特点
战斗主Agent执行战斗规则、状态管理、回合推进、判定胜负不处理叙事，只处理战斗逻辑
敌人Agent（角色扮演型）扮演敌人，根据战场信息用自然语言描述行动意图不直接执行规则，只输出"想要做什么"，由战斗Agent翻译为游戏动作

关键设计：敌人Agent是"演员"而非"裁判"——它只提供意图，战斗Agent负责合法化（检查是否可行、计算结果）。

---

3. 核心 API 设计（极简三接口）

```python
# 1. 推进回合
advance_turn() -> TurnResult
# 推进到下一回合，处理持续效果、刷新行动资源等

# 2. 获取本回合可执行动作
get_available_actions(entity_id) -> List[Action]
# 返回当前实体可执行的动作列表（动态功能子集）
# 关键：根据状态动态过滤（倒地不能移动、沉默不能施法等）

# 3. 执行动作
execute_action(entity_id, action_id, params) -> ActionResult
# 执行指定动作，返回结果 + 新的 available_actions（如果还有剩余资源/动作则非空，空则代表该实体回合结束）
```

---

4. 回合推进流程

```
开始战斗
  ↓
advance_turn() → 确定行动顺序（先攻/敏捷）
  ↓
对当前实体：
  get_available_actions(entity) → 动态动作列表
    ↓
  如果是玩家：展示UI选择
  如果是敌人：将战场状态+动作列表 喂给敌人Agent → 自然语言意图
              → 战斗Agent解析意图匹配到合法Action → execute_action
    ↓
  execute_action 返回新的 available_actions
    ↓
  非空？继续选择动作
  空？下一个实体
    ↓
所有实体行动完毕 → advance_turn() 进入下一回合
```

---

5. 动态功能子集设计（重点）

这是让Agent精准调用的关键。`get_available_actions` 需要基于实时状态动态计算：

过滤维度：
- 资源：剩余行动点/法术位/怒气等
- 状态效果：眩晕（无法行动）、束缚（无法移动）、沉默（无法施法）
- 位置关系：近战武器范围内有无敌人、远程是否被压制
- 冷却/充能：技能是否可用
- 目标合法性：范围内有无有效目标

返回结构示例：

```json
{
  "move": ["forward", "backward", "flank_left"],
  "attack": ["longsword_slash", "shortsword_stab"],
  "spell": ["fireball", "shield"],  // 受法术位限制
  "item": ["healing_potion"],
  "special": ["disengage", "dash"]
}
```

6. 敌人Agent 的 Prompt 设计

```
你是一只地精战士（Goblin Warrior），HP: 15/20，AC: 15。
战场情况：
- 你的位置：东侧岩石后（有掩体）
- 队友：地精弓手（西侧，HP 8/10，正被战士威胁）
- 敌人：玩家战士（前方15尺，HP 25/30）、玩家法师（后方30尺，正在施法）

你当前可执行的动作：
1. 移动：冲到法师身边（会暴露自己）、退回更深掩体、侧移到弓手附近
2. 攻击：用弯刀攻击战士（命中+4，伤害1d6+2）、投掷短矛（射程20/60）
3. 特殊：协助防御（给弓手优势）、尖叫求援（可能引来增援）

用一句话描述你的行动意图，说明理由。
```

敌人Agent输出示例：

> "我压低身子从岩石侧面绕过去，想趁战士不备给他的腿来一刀，然后滚回掩体。"

战斗Agent解析：
- 意图：移动（侧移）+ 攻击（弯刀）+ 移动（退回）
- 检查：移动力足够？攻击在范围内？掩体是否还在？
- 映射到：execute_action(goblin, "move", "flank") + execute_action(goblin, "attack", "scimitar") + execute_action(goblin, "move", "retreat")
